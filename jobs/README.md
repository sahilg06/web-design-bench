# 🚀 `jobs/` — Evaluation Job Execution Outputs

This directory stores the raw execution outputs generated by Harbor during benchmark evaluation runs (`eval/run.py`).

---

## 📂 Directory Layout

Each evaluation run automatically creates a timestamped folder containing the complete execution artifacts for every trial:

```markdown
jobs/2026-07-08__13-32-33/
├── config.json                     # Job configuration & parameters
├── job.log                         # Master execution log from Harbor/Modal
├── result.json                     # Master job status & aggregated statistics
└── v1-fooddeliveryplayful__3mGQAV4/# Individual trial execution folder
    ├── agent/                      # Agent workspace & scratch files
    ├── trial.log                   # Raw execution log for this specific trial
    ├── result.json                 # Trial status, execution time, & token usage
    └── verifier/                   # Verifier execution outputs
        ├── rendered/               # Screenshots & animation frames rendered from agent code
        ├── reward.json             # Final aggregated reward (`blended_reward`, `reward`)
        ├── reward_details.json     # Granular per-page, per-metric breakdown (SSIM, pHash, etc.)
        └── verifier.log            # Execution log from the grading scripts
```

---

## 🖥️ Inspecting Trajectories with Harbor View

You can interactively explore the agent trajectories, tool calls (`Read`, `Write`), DOM differences, and rendered screenshots using Harbor's native web interface.

To launch the viewer on the entire jobs directory:
```bash
harbor view jobs/
```
*Once running, open your web browser to [http://localhost:8080/](http://localhost:8080/) to browse all completed job runs and dive into individual trials.*
