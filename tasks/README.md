# 📦 `tasks/` — Self-Contained Harbor Task Packages

This directory contains the registered, self-contained evaluation task packages generated by `recipe/generate.py`. Each task folder is structured according to the **Harbor Schema v1.1** and contains everything required to evaluate and verify an AI coding agent in an isolated environment.

---

## 📂 Task Directory Structure

```markdown
tasks/v1-aistartupneonhardconfig-73475c/
├── task.toml          # Task metadata, difficulty tier, weights, & container specs (Harbor v1.1)
├── instruction.md     # Prompt presented to the agent (screenshot paths, design directives, rules)
├── environment/
│   ├── Dockerfile     # Isolated container specification (Python 3.11 + Playwright Chromium)
│   └── assets/        # Ground-truth reference screenshots & WebM videos
├── tests/
│   ├── test.sh        # Verifier entry point executed by Harbor
│   ├── grade.py       # CV visual similarity scorer (SSIM + pHash + Color Hist)
│   ├── render.py      # Playwright screenshot & animation frame renderer
│   ├── text_recall.py # Semantic token recall scorer
│   └── pages.json     # Page definitions, viewports, & reward weights
└── solution/
    ├── style.css      # Ground-truth reference stylesheet
    └── *.html         # Ground-truth reference HTML pages
```

---

## 🎛️ Naming Conventions & Task Versions

Task directories are named using the convention: `<recipe_version>-<config_slug>-<seed_hash>`

* **`v1-*` (Part 1: Static Tasks)**: Evaluates static web design replication across 10 diverse archetypes. Contains a single settled reference screenshot (`.png`) per page.
* **`v2-*` (Part 2: Animation Tasks)**: Evaluates dynamic CSS animations and temporal state adherence across 4 archetypes. Contains a settled screenshot (`.png`), 3-4 frozen animation frames (`_t0.png`, `_t500.png`, `_t1200.png`, `_t1800.png`), and a 3-second video (`.webm`) per page.
* **`v3-*` (Part 3: Framework Tasks)**: Evaluates multi-page Single Page Applications (SPAs) built with React JS or Solid JS and styled with Vanilla CSS or Tailwind CSS. Contains a complete Vite project structure (`package.json`, `vite.config.js`, `src/`) and evaluates SPA tab-switching navigation.

---

## 🛡️ Isolation & Reproducibility

Every task in this directory is completely decoupled from the rest of the repository once generated. The grading scripts (`grade.py`, `render.py`, `text_recall.py`) and environment definitions (`Dockerfile`) are self-contained within each task package. This ensures that tasks can be shared, archived, or executed on remote sandboxes (like Modal) with 100% reproducibility.
